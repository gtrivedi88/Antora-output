1. Scope

This document covers the sanity tests contained in HACBS that are executed as part of the HACBS component build pipeline. The sanity tests for HACBS automatically test all application images to ensure they are up to date, correctly formatted, and protect them from security vulnerabilities.

2. Sanity Tests

The HACBS component build pipeline supports several types of tests, including sanity tests. The sanity tests used in HACBS are executed in the form of Tekton tasks. The utility used for validating container information is conftest. Currently implemented sanity tests are listed below. Only sanity tests which are currently implemented are listed in this section.
