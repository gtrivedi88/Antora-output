These rules are applied to Tekton pipeline definitions.

1. Pipeline definition sanity checks

Currently there is just a check to confirm the input appears to be a Pipeline definition. We may add additional sanity checks in future.

  • Package name: basic

  • Package full path: policy.pipeline.basic

1.1. Input data has unexpected kind

A sanity check to confirm the input data has the kind "Pipeline"

  • Rule type: FAILURE

  • Failure message: Unexpected kind '%s'

  • Code: unexpected_kind

  • Source

2. Required tasks

HACBS expects that certain Tekton tasks are executed during image builds. This package includes policy rules to confirm that the pipeline definition includes the required Tekton tasks.

  • Package name: required_tasks

  • Package full path: policy.pipeline.required_tasks

2.1. No tasks in Pipeline

This policy enforces that at least one Task is present in the Pipeline definition.

  • Rule type: FAILURE

  • Failure message: No tasks found in Pipeline definition

  • Code: tasks_missing

  • Source

2.2. Missing required task

This policy enforces that the required set of tasks are included in the Pipeline definition.

  • Rule type: FAILURE

  • Failure message: Required task %q is missing

  • Code: missing_required_task

  • Source

2.3. Missing future required task

This policy warns when a task that will be required in the future was not included in the Pipeline definition.

  • Rule type: WARNING

  • Warning message: Task %q is missing and will be required in the future

  • Code: missing_future_required_task

  • Source

2.4. Missing required tasks data

The policy rules in this package require the required-tasks data to be provided.

  • Rule type: FAILURE

  • Failure message: Missing required task-bundles data

  • Code: missing_required_data

  • Source

3. Task bundle

To be able to reproduce and audit builds accurately it’s important to know exactly what happens during the build. To do this Enterprise Contract requires that all tasks are defined in a set of known and trusted task bundles. This package includes rules to confirm that the tasks in a Pipeline definition are defined in task bundles, and that the task bundles are from the list of known and trusted bundles.

  • Package name: task_bundle

  • Package full path: policy.pipeline.task_bundle

3.1. Task bundle was not used or is not defined

Check for existence of a task bundle. Enforcing this rule will fail the contract if the task is not called from a bundle.

  • Rule type: FAILURE

  • Failure message: Pipeline task '%s' does not contain a bundle reference

  • Code: disallowed_task_reference

  • Source

3.2. Task bundle reference is empty

Check for a valid task bundle reference being used.

  • Rule type: FAILURE

  • Failure message: Pipeline task '%s' uses an empty bundle image reference

  • Code: empty_task_bundle_reference

  • Source

3.3. Unpinned task bundle reference

Check if the Tekton Bundle used for the Tasks in the Pipeline definition is pinned to a digest.

  • Rule type: WARNING

  • Warning message: Pipeline task '%s' uses an unpinned task bundle reference '%s'

  • Code: unpinned_task_bundle

  • Source

3.4. Task bundle is out of date

Check if the Tekton Bundle used for the Tasks in the Pipeline definition is the most recent acceptable one. See the list of acceptable task bundles at Acceptable Bundles or look at data/acceptable_tekton_bundles.yml in this git repository. The meaning of an acceptable bundle is explained in Acceptable Bundles

  • Rule type: WARNING

  • Warning message: Pipeline task '%s' uses an out of date task bundle '%s'

  • Code: out_of_date_task_bundle

  • Source

3.5. Task bundle is not acceptable

Check if the Tekton Bundle used for the Tasks in the Pipeline definition are acceptable given the tracked effective_on date. See the list of acceptable task bundles at Acceptable Bundles or look at data/acceptable_tekton_bundles.yml in this git repository. The meaning of an acceptable bundle is explained in Acceptable Bundles

  • Rule type: FAILURE

  • Failure message: Pipeline task '%s' uses an unacceptable task bundle '%s'

  • Code: unacceptable_task_bundle

  • Source

3.6. Missing required data

The policy rules in this package require the task-bundles data to be provided.

  • Rule type: FAILURE

  • Failure message: Missing required task-bundles data

  • Code: missing_required_data

  • Source
